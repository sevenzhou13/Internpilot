# 项目记忆：有界上下文工作法

最后更新：2026-07-11

## 方法

本项目采用卡帕西式的工程记忆原则：把会反复用到的事实固化为短小、可验证的仓库文件；每个任务只加载相关上下文；让代码、测试和文档一起积累，而不是依赖某一段聊天记录。

这不是把所有历史塞进提示词，而是三层记忆：

1. **稳定事实**：`README.md`、需求、架构和 ADR。只在事实或决策变化时修改。
2. **工作记忆**：`context/HANDOFF.md`。只保留当前基线、已验证结果、阻塞点和下一步，目标是不超过一屏。
3. **证据历史**：进度、测试报告、Git 历史。按事件追加，供需要时检索，不作为每次任务的默认上下文。

## 当前事实

- 主架构为 FastAPI + React CDN + SQLite；`server.py` 是服务入口。
- 个人真实数据仅保存在本地；公开 Demo 必须使用匿名数据。
- 岗位导入支持文本、文件、公开 URL 和 Chrome Clipper，均先预览、结构化和去重。
- 岗位类别采用 AI 驱动的开放 taxonomy；当前八类只是规则兜底和输入示例。旧 TF-IDF + Logistic Regression 仅保留为兼容实验，非产品或课程主线。
- 匹配仍以可解释规则为主；本地 TF-IDF RAG、岗位分析控制台和 TF-IDF + K-Means 聚类已完成。中文向量检索与语义匹配仍是后续增强项。
- Demo 模式使用独立匿名种子数据库并禁用外部 LLM；课程实验通过脚本导出聚合统计、固定种子聚类配置与指标，不导出原始 JD 或个人数据。
- 课程训练数据可在用户明确授权下低频采集公开 JD；原始数据必须保存在 Git 忽略的 `data/raw/`，并保留来源、时间、查询条件和人工复核状态。

## 上下文装载协议

开始任务时依次读取根目录 `README.md`、本文件、`HANDOFF.md`，再读取与任务直接相关的权威文档和代码。不要阅读 `archive/`，除非任务明确要求追溯历史。

完成任务时：

1. 运行与风险相称的测试。
2. 更新 API、数据字典、架构、ADR、进度或测试报告中受到影响的部分。
3. 将当前可行动状态压缩写入 `HANDOFF.md`，删除已过期的临时细节。
4. 运行 `git diff --check`；不得提交密钥或个人数据。

## 文档冲突处理

当前代码和本目录的权威文档优先于旧计划。发现冲突时，先记录差异和决定；不要在未确认的情况下按旧 Streamlit 文档重写现有 FastAPI 应用。
