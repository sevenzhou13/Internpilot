# 重要设计决策记录

## ADR-001：保留当前 FastAPI + React 架构

- 日期：2026-07-10
- 决策：不恢复旧 Streamlit 路线，在现有 FastAPI、React CDN 和 SQLite 上增量开发。
- 原因：当前仓库已经实现完整前后端和插件流程，重写会损失可用功能并增加六周交付风险。

## ADR-002：数据库采用显式迁移和升级前备份

- 日期：2026-07-10
- 决策：使用 `schema_migrations` 记录版本，旧数据库首次升级前复制到 `data/backups/`。
- 原因：真实个人数据不可通过删除重建解决，迁移必须可追踪、可回退。

## ADR-003：岗位方向与模型类别分离

- 日期：2026-07-10
- 决策：保留面向个人偏好的 `role_type`，新增更广的 `job_category` 用于模型和市场分析。
- 原因：避免课程分类体系破坏已有求职偏好、筛选和 Prompt 行为。

## ADR-004：不训练缺少真实标签的投递预测模型

- 日期：2026-07-10
- 决策：第一个课程版本训练岗位分类模型，匹配使用可解释规则与语义相似度；投递结果模型等待真实事件积累。
- 原因：把删除、未处理等行为直接当负样本会产生无法证明的伪精度。

## ADR-005：以有界项目记忆替代聊天和旧计划作为事实来源

- 日期：2026-07-10
- 决策：当前需求、架构、ADR、交接和测试证据分别维护在 `docs/` 的权威文档中；旧 Streamlit 方案迁入 `docs/archive/legacy-streamlit/`，不再作为开发依据。
- 原因：项目曾经历架构迁移，根目录旧文档会诱导后续智能体回到 Streamlit。短小的工作记忆加可检索的历史证据，能减少上下文噪声并让交接可重复。

## ADR-006：为腾讯招聘域名保留受限的测试代理兼容

- 日期：2026-07-10
- 决策：URL 导入仅允许 `join.qq.com` 使用被本地代理映射到 `198.18.0.0/15` 的地址；其他非全局 IP 继续拒绝。
- 原因：该映射在当前环境中会误伤正常公开招聘链接。按域名和精确地址段双重限制，可以恢复导入同时保持 SSRF 防护边界。
