# 重要设计决策记录

## ADR-001：保留当前 FastAPI + React 架构

- 日期：2026-07-10
- 决策：不恢复旧 Streamlit 路线，在现有 FastAPI、React CDN 和 SQLite 上增量开发。
- 原因：当前仓库已经实现完整前后端和插件流程，重写会损失可用功能并增加六周交付风险。

## ADR-002：数据库采用显式迁移和升级前备份

- 日期：2026-07-10
- 决策：使用 `schema_migrations` 记录版本，旧数据库首次升级前复制到 `data/backups/`。
- 原因：真实个人数据不可通过删除重建解决，迁移必须可追踪、可回退。

## ADR-003：岗位方向与模型类别分离

- 日期：2026-07-10
- 决策：保留面向个人偏好的 `role_type`，新增更广的 `job_category` 用于模型和市场分析。
- 原因：避免课程分类体系破坏已有求职偏好、筛选和 Prompt 行为。

## ADR-004：不训练缺少真实标签的投递预测模型

- 日期：2026-07-10
- 决策：第一个课程版本训练岗位分类模型，匹配使用可解释规则与语义相似度；投递结果模型等待真实事件积累。
- 原因：把删除、未处理等行为直接当负样本会产生无法证明的伪精度。
