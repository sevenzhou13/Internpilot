# 重要设计决策记录

## ADR-001：保留当前 FastAPI + React 架构

- 日期：2026-07-10
- 决策：不恢复旧 Streamlit 路线，在现有 FastAPI、React CDN 和 SQLite 上增量开发。
- 原因：当前仓库已经实现完整前后端和插件流程，重写会损失可用功能并增加六周交付风险。

## ADR-002：数据库采用显式迁移和升级前备份

- 日期：2026-07-10
- 决策：使用 `schema_migrations` 记录版本，旧数据库首次升级前复制到 `data/backups/`。
- 原因：真实个人数据不可通过删除重建解决，迁移必须可追踪、可回退。

## ADR-003：岗位方向与模型类别分离

- 日期：2026-07-10
- 决策：保留面向个人偏好的 `role_type`，新增更广的 `job_category` 用于模型和市场分析。
- 原因：避免课程分类体系破坏已有求职偏好、筛选和 Prompt 行为。

## ADR-004：不训练缺少真实标签的投递预测模型

- 日期：2026-07-10
- 决策：第一个课程版本训练岗位分类模型，匹配使用可解释规则与语义相似度；投递结果模型等待真实事件积累。
- 原因：把删除、未处理等行为直接当负样本会产生无法证明的伪精度。

## ADR-005：以有界项目记忆替代聊天和旧计划作为事实来源

- 日期：2026-07-10
- 决策：当前需求、架构、ADR、交接和测试证据分别维护在 `docs/` 的权威文档中；旧 Streamlit 方案迁入 `docs/archive/legacy-streamlit/`，不再作为开发依据。
- 原因：项目曾经历架构迁移，根目录旧文档会诱导后续智能体回到 Streamlit。短小的工作记忆加可检索的历史证据，能减少上下文噪声并让交接可重复。

## ADR-006：为腾讯招聘域名保留受限的测试代理兼容

- 日期：2026-07-10
- 决策：URL 导入仅允许 `join.qq.com` 使用被本地代理映射到 `198.18.0.0/15` 的地址；其他非全局 IP 继续拒绝。
- 原因：该映射在当前环境中会误伤正常公开招聘链接。按域名和精确地址段双重限制，可以恢复导入同时保持 SSRF 防护边界。

## ADR-007：允许用户授权下的低频公开 JD 采集

- 日期：2026-07-11
- 决策：为构建课程训练集，可在用户明确授权下采集无需登录、无需绕过限制的公开岗位 JD，并记录来源、时间、查询条件和失败原因。
- 边界：不绕过验证码、登录、Cookie、访问控制或反爬机制；遇到平台限制立即停止该来源采集并改用用户合法导出或其他公开来源。
- 原因：课程分类模型需要足量、可追溯的数据。低频且可审计的公开采集可降低手工录入成本，同时保留合规和安全边界。

## ADR-008：规则引导的全量类别建议不作为训练真值

- 日期：2026-07-11
- 决策：可使用已有规则标签训练临时分类器，为无类别公开 JD 批量生成建议；建议写入模型预测历史和复核 CSV，不写入 `job_category_labels`。
- 原因：这能减少人工浏览成本，但规则类别会产生类别偏置。必须以低置信度排序、外部类别标签映射或人工复核完成真值闭环，不能把自训练建议当作独立监督信号。

## ADR-009：正式课程训练优先使用带原生类别的中文 JD 数据

- 日期：2026-07-11
- 决策：不将仅带学历标签的中文 JD 及其规则/LLM 推断类别作为正式训练集或课程指标来源。优先取得许可明确、同时包含岗位名称、JD 和原始 `category` 的中文数据；经版本化八类映射后以 `external_dataset` 导入。
- 原因：类别字段是可追溯监督信号，中文 JD 与产品使用场景一致；从标题或正文推断出的标签会把模型自身偏差循环写回训练数据。外部类别映射的评估仍只能表述为“外部数据集标签评估”。

## ADR-010：岗位类别采用 AI 驱动的开放 taxonomy

- 日期：2026-07-11
- 决策：产品不再将当前八类岗位大类作为固定 taxonomy，也不把训练分类器作为主流程。LLM 可基于岗位名称和 JD 提出新类别、合并或拆分建议；用户确认后写入岗位。当前八类仅用于无 LLM 时的规则兜底和输入提示。
- 边界：AI 不得静默批量改写历史类别。分类依据和 taxonomy 建议写入岗位的结构化解析结果；人工确认优先于 AI 和规则结果。旧 `TF-IDF + Logistic Regression` 接口仅作为兼容实验保留。
- 原因：类别是求职分析的辅助视图，开放且可演进的 taxonomy 比维护低价值、易过时的固定分类器更适合产品；课程主线转为 JD 清洗、技能分析、文本聚类和可解释匹配。

## ADR-011：公开展示使用隔离匿名 Demo

- 日期：2026-07-11
- 决策：`APP_MODE=demo` 使用独立数据库并自动写入代码内置的匿名种子数据，强制禁用外部 LLM 调用；Render 配置只部署该模式。
- 原因：公开演示需要可重复的初始状态，但不能暴露本地个人数据库、真实 JD、简历或 API Key。Demo 写入与个人本地模式隔离，云端不承诺持久化。

## ADR-012：公开 URL 默认使用本地可见浏览器解析

- 日期：2026-07-17
- 决策：本地模式下，前端粘贴的单个公开 URL 默认交给 Playwright 控制的专用可见 Chrome 渲染；旧 HTTP 抓取接口仅作兼容。浏览器使用独立 Git 忽略配置目录，一次只运行一个任务，并校验初始及跳转地址。
- 边界：不复用用户日常 Chrome 配置，不复制或上传 Cookie，不隐藏浏览器，不自动处理登录或验证码，不批量采集；Demo 模式禁用。
- 原因：大量招聘官网依赖 JavaScript、登录态或浏览器验证，服务端 HTTP 客户端即使网络正常也只能取得中间页。可见浏览器能保留直接粘贴链接的体验，同时让用户掌控验证过程。
