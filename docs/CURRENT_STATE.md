# 原项目功能审计

审计日期：2026-07-10

## 已具备能力

- FastAPI 提供静态前端和 JSON/SSE API。
- React CDN 前端包含控制台、岗位、经历、简历、面试、AI 助手和偏好页面。
- SQLite 已覆盖偏好、教育、经历、岗位、简历和生成历史。
- 岗位支持手动输入、粘贴 JD 智能解析和 Clipper 保存。
- 匹配算法能组合技能、经历、岗位要求和地点，但仍基于字符串/规则。
- 简历、面试和聊天支持 OpenAI 兼容接口及流式输出。
- Manifest V3 插件能读取页面、调用本地 JD 解析并保存岗位。

## 与六周目标的差距

- 缺少正式测试、数据库版本迁移和可隔离测试数据库。
- JD 结构化结果只写回少量字段，没有标准化技能、薪资、学历和去重。
- `role_type` 只有个人求职方向，尚无课程分类模型使用的岗位大类。
- 没有可复现的数据清洗、训练、评估、模型版本和人工复核流程。
- 匹配没有语义向量、分项解释和人工排序评估。
- AI 上下文仍直接拼接，没有知识片段索引、检索和来源引用。
- 控制台只有基础统计，没有技能缺口、投递漏斗、分类和聚类分析。
- 当前公开部署方案与旧 Streamlit 文档不一致，且没有 Demo 数据隔离。

## 已发现并修复的基线问题

- 个人偏好 API 接受专业、学校、学历和求职类型，但旧数据库写入函数没有保存这些字段。
- 数据库路径硬编码，无法安全运行测试。
- 数据库升级依靠捕获所有异常的临时 `ALTER TABLE`，没有版本和备份。
- CORS 默认允许任意来源。

## 暂存风险

- 前端通过 CDN 加载 React、Babel 和字体，离线环境不可用。
- `server.py` 体积较大，后续新增接口应逐步拆分路由，但不做无关大重构。
- 当前 API 对不存在资源的更新/删除通常仍返回成功，后续按接口逐步补 404。
- 当前自动化覆盖率为基线而非目标，生成、导入和插件仍需专项测试。
