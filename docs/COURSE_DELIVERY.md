# 数据挖掘课程交付材料

最后更新：2026-07-11

## 课题表述

**基于招聘 JD 的岗位需求画像、文本聚类与可解释求职匹配系统。**

研究对象是用户导入或授权采集的 JD。系统将文本清洗为技能、城市、学历和经验等结构化特征；随后以 TF-IDF + K-Means 聚类识别需求簇，并以技能频次、投递漏斗和技能差距完成描述性分析；最后用可解释规则输出求职匹配证据。

## 可展示的算法与证据

| 环节 | 方法 | 可交付证据 |
|---|---|---|
| 数据准备 | 文本清洗、标准技能抽取、URL/字段去重 | 导入预览、结构化字段、去重规则 |
| 描述性分析 | 频次统计、漏斗分析、技能差距 | 控制台聚合卡片、实验报告 JSON/Markdown |
| 无监督学习 | TF-IDF `(1,2)-gram` + K-Means | 候选簇数、固定 `random_state=42`、silhouette、关键词和代表岗位 |
| 应用解释 | 技能/经历/要求/地点加权匹配 | 分项得分、匹配技能和待补技能 |
| 检索 | 本地 TF-IDF 文本检索 | 来源可追溯的知识片段 |

AI 岗位类别是产品的开放 taxonomy 辅助能力，不作为课程模型的性能结论。

## 生成实验结果

```powershell
# 先在匿名 Demo 或本地匿名数据上启动并确认数据库内容
.\.venv\Scripts\python.exe scripts\run_mining_experiment.py --max-clusters 5
```

命令在 Git 忽略的 `data/processed/mining_experiment/` 下生成：

- `report.json`：可机读的配置、聚合统计和聚类指标；
- `report.md`：可直接作为报告结果章节的初稿。

提交课程作业时仅提交匿名样例、脚本、配置和聚合结果；不要提交原始 JD、个人经历、数据库或 API Key。

## 三分钟答辩流程

1. 说明数据边界与匿名化原则，展示导入后结构化和去重。
2. 打开控制台，说明高频技能、投递漏斗和技能差距的业务含义。
3. 展示自动选择簇数、silhouette 和每个需求簇的关键词/代表岗位。
4. 打开岗位详情，展示四项可解释匹配证据；强调它不是黑箱“适合/不适合”判断。
5. 说明复现实验命令、固定随机种子和限制条件。
