# InternPilot 架构基线

最后更新：2026-07-11

## 当前结构

```text
Chrome Clipper ─┐
文件/文本/URL ──┼─> FastAPI API ─> SQLite
React CDN 前端 ─┘       │
                        ├─> 可解释规则匹配
                        ├─> AI 开放岗位类别（已实现）
                        ├─> 岗位聚类（TF-IDF + K-Means）
                        ├─> 本地 RAG 检索（TF-IDF 基线）
                        └─> OpenAI 兼容 LLM
```

- `server.py`：HTTP API、SSE 流式输出和静态页面入口。
- `modules/db.py`：SQLite 访问和版本迁移的唯一入口。
- `modules/config.py`：运行模式、数据库路径和跨域来源配置。
- `modules/matcher.py`：可解释规则匹配与分项证据；后续接入语义维度。
- `modules/llm_client.py`：OpenAI 兼容模型配置和调用封装。
- `static/`：不需要构建步骤的 React 页面。
- `clipper/`：Manifest V3 浏览器插件。

## 运行模式

- `local`：默认模式，完整读写，数据库位于 `data/internpilot.db`。
- `test`：自动化测试模式，数据库路径由测试临时目录指定。
- `demo`：使用独立的 `data/internpilot_demo.db`，首次启动自动写入匿名种子数据，并强制禁用外部 AI 调用；不读取本地个人数据库。

## 数据与模型流

1. 岗位输入先生成预览，不直接写库。
2. 规则清洗和 LLM 抽取共同生成结构化岗位。
3. 去重通过规范化字段和原文哈希完成。
4. LLM 解析可给出开放的岗位类别、分类依据和 taxonomy 优化建议；当前八类仅是规则兜底与输入示例。人工确认优先于 AI 和规则结果。用户确认类别合并时，旧值写入 `job_category_history`；旧的 TF-IDF 分类器只作兼容实验，不能限制产品 taxonomy。
5. 匹配模块组合规则与偏好，并返回技能、经历、硬性要求和地点的分项证据；语义维度是下一阶段工作。
6. RAG 已对岗位、经历和简历切片并使用 TF-IDF 检索；相关片段会注入 AI 助手上下文。中文向量检索是后续增强项。
7. 分析接口与控制台当前提供岗位类别、来源、投递漏斗、高频技能、技能短板和岗位文本聚类；中文向量聚类是后续增强项。
8. 课程实验导出只输出聚合统计、聚类配置、silhouette、关键词和代表岗位标题；Demo 与公开部署不输出真实个人数据或原始 JD。

前端岗位详情展示规则匹配分项、匹配技能和待补足技能；AI 助手会显示本轮问答检索到的本地资料来源。

## 兼容原则

- 已有 API 和字段不删除；新增结构通过可空字段和新表扩展。
- `jobs.status` 保留为当前状态，投递事件表记录历史变化。
- RAG 或模型不可用时回退到当前规则和直接上下文，不阻断基础功能。
