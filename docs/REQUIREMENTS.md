# InternPilot 二次开发需求基线

最后更新：2026-07-10

## 1. 双重目标

1. 《数据挖掘》课程项目：必须具备可复现的数据准备、模型训练、指标评估、误差分析和可视化结果，不能只调用大模型 API。
2. 真实个人工具：必须能帮助项目所有者管理岗位、判断投递优先级、生成基于真实经历的材料，并保持本地个人数据安全。

## 2. 已确认产品边界

- 保留 FastAPI + React CDN + SQLite 架构。
- 本地模式保存真实个人数据；公开 Demo 只使用匿名示例数据。
- 保留现有 `role_type` 求职方向，同时增加用于课程分析的 `job_category` 岗位类别。
- 训练传统分类基线和“预训练中文向量 + 轻量 MLP”模型；不把完整 Transformer 微调列为必做项。
- 不在缺少真实标签时训练投递结果预测模型；先记录投递事件和人工反馈。
- 不实现商业招聘平台批量爬虫、验证码绕过、Cookie 采集或反爬规避。

## 3. 六周验收里程碑

- 第 1 周：应用可复现启动，数据库可迁移，自动化测试和交接文档建立。
- 第 2 周：文本、文件、公开 URL 均可预览导入，JD 可结构化并去重。
- 第 3 周：岗位分类数据流水线、训练脚本、人工复核集和模型卡完成。
- 第 4 周：可解释混合匹配和 RAG 接入，形成课程答辩 MVP。
- 第 5 周：聚类、技能差距、投递漏斗和岗位分析视图完成。
- 第 6 周：匿名 Demo、Render 部署、安全检查、课程和作品集材料完成。

## 4. 质量要求

- 所有数据库变更必须可重复执行，并在升级旧库前创建备份。
- 新增 API 必须有成功、校验失败和降级路径测试。
- 模型训练必须固定随机种子，保存配置、版本、指标和标签映射。
- AI 输出必须说明由 AI 生成，并能追溯使用的经历或岗位片段。
- 仓库不得包含密钥、真实简历、真实联系方式或真实个人数据库。
